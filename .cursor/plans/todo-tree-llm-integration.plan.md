# Plan: Todo Tree + LLM (robocza specyfikacja)

## Cel

Ujednolicic tagi TODO w kodzie, tak aby:

- uzytkownik szybko znajdowal krytyczne miejsca,
- LLM mial stabilne punkty odniesienia,
- w przyszlosci dalo sie automatycznie generowac raporty i podsumowania.

## Standard tagow i format

### Tagi

`TODO`, `FIXME`, `BUG`, `HACK`, `NOTE`, `OPTIMIZE`, `PERF`, `SECURITY`

### Format komentarzy (zalecany)

```
// TODO[area=auth,priority=high]: Przepisac logike sesji na v2.
// FIXME[area=server,priority=medium]: Poprawic mapowanie DTO.
// NOTE[area=ui,priority=low]: Ten komponent zalezy od Nuxt UI.
```

Klucze metadanych (opcjonalne):

- `area`: `ui|content|domain|server|auth|forms|sections|tooling`
- `priority`: `low|medium|high`

## Konfiguracja Todo Tree (workspace)

Dodac do `.vscode/settings.json`:

```
"todo-tree.general.tags": ["TODO","FIXME","BUG","HACK","NOTE","OPTIMIZE","PERF","SECURITY"],
"todo-tree.highlights.defaultHighlight": {
  "icon": "alert",
  "type": "text",
  "foreground": "red",
  "background": "white",
  "opacity": 50,
  "iconColour": "blue"
},
"todo-tree.highlights.customHighlight": {
  "TODO": { "icon": "check", "type": "line" },
  "FIXME": { "foreground": "black", "iconColour": "yellow", "gutterIcon": true },
  "SECURITY": { "foreground": "white", "background": "red", "type": "line" }
}
```

## Raport TODO (wariant A, lekki)

### Cel

Generowac raport z TODO i przekazywac go LLM.

### Komenda

```
rg --no-heading -n "(TODO|FIXME|BUG|HACK|NOTE|OPTIMIZE|PERF|SECURITY)" > reports/todo-tree.md
```

### Uzycie z LLM

1. Wygeneruj raport.
2. Dodaj raport jako kontekst do zapytania.
3. Popros o streszczenie i propozycje kolejnych krokow.

## Integracja LLM (wariant B, glebszy)

### Cel

Automatyczne podsumowanie TODO, priorytety, wskazanie powiazanego kontekstu.

### Wymagania

- Konsekwentny format tagow z metadanymi.
- Raport w formacie markdown lub json (do dalszej obrobki).

### Kroki rozwoju

1. Standaryzacja tagow w kodzie.
2. Skrypt generujacy raport (np. `reports/todo-tree.md`).
3. Pipeline: raport -> LLM -> summary + rekomendacje.
4. (Opcjonalnie) mapowanie TODO do moduow/komponentow.

## Ograniczenia dostepu

LLM nie ma bezposredniego dostepu do rozszerzenia VS Code.
Najprostszy sposob integracji to eksport raportu i przekazanie go LLM jako kontekstu.
